Module

Array

import collections.array

Tableaux generiques pour le code utilisateur. Importer collections.array donne acces a Array[T], Array.empty[T](), Array.fill[T], Array.tabulate[T] et Array.range. Les anciens noms ArrayFill[T] et ArrayRange restent disponibles pour compatibilite. Les types bas niveau comme IntArray, ArrayInt, ObjectArray[T] et ArrayObject[T] sont des details de representation dans les cas courants : ils peuvent apparaitre dans certains retours ou diagnostics, mais les exemples applicatifs devraient nommer Array[T].

class Recommandee

Array

class Array[T](size: Int)

Tableau mutable de taille fixe.

Array[T] est le nom public a utiliser. Le compilateur choisit la representation adaptee selon T.

Cree un tableau rempli avec la meme valeur.

Preferer Array.fill[T](size, value) dans le nouveau code.

def Recommandee

empty

def Array.empty[T](): Array[T]

Cree un tableau vide.

La representation concrete peut encore etre ArrayObject[T] pour les types generiques. Le code utilisateur doit la manipuler comme un Array[T] via les operations communes.

def Recommandee

fill

def Array.fill[T](size: Int, value: T): Array[T]

Cree un tableau rempli avec la meme valeur.

Pour T = Int, Long, Float, Double ou Bool, le compilateur choisit une specialisation primitive. Les autres types utilisent la representation generique actuelle, sans que le code utilisateur doive la nommer.

def Recommandee

tabulate

def Array.tabulate[T](size: Int, f: (Int) => T): Array[T]

Cree un tableau en calculant chaque element depuis son index.

Comme Array.fill, cette fabrique garde le nom public Array[T] meme si la representation visible est specialisee ou generique selon T.
