Module

Array

import collections.array

Tableaux generiques pour le code utilisateur. Importer collections.array donne acces a Array[T], Array.empty[T](), Array.fill[T], Array.tabulate[T] et Array.range. Les anciens noms ArrayFill[T] et ArrayRange restent disponibles pour compatibilite. Les types bas niveau comme IntArray, ArrayInt, ObjectArray[T] et ArrayObject[T] sont des details de representation dans les cas courants : les exemples applicatifs devraient nommer Array[T].

Array

class Array[T](size: Int)

Tableau mutable de taille fixe.

Array[T] est le nom public a utiliser. Le compilateur choisit la representation adaptee selon T.

def Recommandee

Array

def Array[T].sorted(lessThan: (T, T) => Bool): Array[T]

Retourne une copie triee sans modifier le tableau source.

La V0 demande un predicat d'ordre strict utilisateur, par exemple `xs.sorted((left, right) => left.score() < right.score())`. Les tableaux primitifs numeriques exposent aussi `sorted()` avec leur ordre naturel.

def Recommandee

fill

def Array.fill[T](size: Int, value: T): Array[T]

Cree un tableau rempli avec la meme valeur.

Pour T = Int, Long, Float, Double ou Bool, le compilateur choisit une specialisation primitive. Les autres types utilisent la representation generique actuelle, sans que le code utilisateur doive la nommer.

def Recommandee

fill

def Array.fill[T](n: Int)(elem: => T): Array[T]

Cree un tableau en reevaluant l'expression fournie pour chaque element.

La forme curryfiee accepte un parametre par nom: elem est evalue une fois par case, de gauche a droite, ce qui permet de produire des valeurs differentes a chaque iteration.

def Recommandee

tabulate

def Array.tabulate[T](size: Int, f: (Int) => T): Array[T]

Cree un tableau en calculant chaque element depuis son index.

Comme Array.fill, cette fabrique garde le nom public Array[T] meme si le backend est specialise ou generique selon T.
